25/*
26 * This file describes a simple C API for Faust objects including the audio drivers.
27 * Faust objects are built with createDsp, then initialized with initDsp and the sampling rate and buffer size.
28 * Then start() is called to open the drivers and start processing audio until stop() is called.
29 * All parameters can be accessed and controlled (typically using getParamsCountDsp/setParamValueDsp/getParamValueDsp).
30 *
31 * To be compiled with something like:
32 * c++ -std=c++11 -DLLVM_DSP -DPORTAUDIO_DRIVER -DSOUNDFILE faust-dynamic-engine.cpp
33 * and linked with libfaust (either static or dynamic version, libsndfile (https://github.com/libsndfile/libsndfile)
34 * and PortAudio (https://www.portaudio.com).
35 *
36 * The Interpreter backend can possibly be used instead of LLVM, using INTERP_DSP compilation flag as in:
37 * c++ -std=c++11 -DINTERP_DSP -DPORTAUDIO_DRIVER -DSOUNDFILE faust-dynamic-engine.cpp
38 *
39 * Several different audio drivers can possibly be compiled (to be chosen at init time). With something like:
40 * c++ -std=c++11 -DLLVM_DSP -DPORTAUDIO_DRIVER -DRTAUDIO_DRIVER -DJACK_DRIVER -DSOUNDFILE faust-dynamic-engine.cpp
41 * and adding the needed libraries at link time.
42 *
43 * JACK (https://jackaudio.org) can possibly be used instead of PortAudio to provide a "multiple connectable DSP" experience:
44 * each DSP is created with createDsp/initDsp, then connected to other with getNumInputsDsp/getNumOutputsDsp and
45 * connectDsp/disconnectDsp/isConnectedDsp functions, or using an external connection tool
46 * like QjackCtl (https://qjackctl.sourceforge.io).
47 */
